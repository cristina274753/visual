Gestión de Equipos Deportivos

Enunciado

En este ejercicio, vamos a desarrollar un programa que permita gestionar equipos deportivos. Cada equipo puede tener un entrenador y varios jugadores. Deberás implementar clases para cada entidad (Equipo, Entrenador y Jugador), y gestionar la interacción entre ellas.

Requisitos

  1. Crear tres clases: Equipo, Entrenador y Jugador.
    • Equipo: Cada equipo tiene un ID único, nombre, deporte y puede tener un entrenador y varios jugadores.
    • Entrenador: Cada entrenador tiene un ID único, nombre y correo electrónico.
    • Jugador: Cada jugador tiene un ID único, nombre, posición y dorsal.
  2. Validar los datos ingresados:
    • No permitir IDs duplicados para equipos, entrenadores o jugadores.
    • Verificar que el ID del entrenador exista antes de asignarlo a un equipo.
    • Permitir agregar múltiples jugadores a un equipo.
  3. Implementar las siguientes funcionalidades dinámicas:
    • Registrar equipos, entrenadores y jugadores mediante formularios.
    • Asignar un entrenador y jugadores a un equipo.
    • Mostrar listados dinámicos en tablas para equipos, entrenadores y jugadores registrados.
    • Mostrar mensajes de error o éxito en tiempo real.
